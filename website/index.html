About this project

RAG (retrieval-augmented generation) is a foundational pattern behind many AI tools and assistants — but most tutorials hide the interesting parts behind framework calls. tiny-rag-lab is my attempt to build every stage by hand: document loading, chunking, embedding, local vector search, retrieval, prompt assembly, answer generation, evaluation, and failure diagnosis.

The goal is not a production platform. The goal is deep, inspectable understanding — readable code over framework magic, evaluation before optimization, failure analysis before advanced features.

Built across nine incremental phases, the project now covers classic dense retrieval, BM25 keyword search, hybrid fusion, reranking, answer quality judging, context budgeting, and structural/semantic chunking — all wired through a clean Python CLI with trace output at every stage.

What it covers

Built phase-by-phase, each capability measurable and debuggable.

⬡

Classic RAG baseline

Full pipeline: document loading, text normalization, chunking, embedding, local cosine-similarity retrieval, grounded prompt assembly, answer generation, and citations.

⬡

Retrieval mechanics

Dense vector search, BM25 keyword retrieval, and hybrid fusion via Reciprocal Rank Fusion — all configurable and comparable with a single CLI flag.

⬡

Evaluation harness

rag eval measures retrieval quality (hit rate, MRR, context precision/recall) and answer quality (faithfulness, relevance, correctness) against a prepared QA set.

⬡

Observability & failure lab

Per-query trace output exposes retriever, scores, latency, and prompt context. rag diagnose runs curated failure cases with baseline vs. intervention comparison.

⬡

Reranking & answer judging

Optional second-pass reranking (fake or cross-encoder). LLM-as-judge interface for answer-quality verdicts — both fakeable for offline testing.

⬡

Context budget & chunking

Token-budget context packing with traceable omitted chunks. Three chunking strategies: fixed-character, Markdown-structural, and experimental semantic topic-shift.

A clean CLI interface

Index a corpus, retrieve evidence, ask a question — each step inspectable.

terminal 
# Build the index from a local corpus
$ rag index --corpus corpus/watsonx-docsqa --chunking-strategy structural

# Retrieve top-5 chunks with hybrid search
$ rag retrieve "What is a deployment space?" --retriever hybrid --top-k 5

# Ask a question and get a grounded answer with citations
$ rag ask "How do I deploy a model?" --context-budget 8192

# Evaluate retrieval quality against a QA set
$ rag eval --qa-file corpus/watsonx-docsqa/qa.jsonl --retriever hybrid

# Diagnose retrieval failure cases
$ rag diagnose --cases-file tests/fixtures/failure/cases.jsonl
